Design Direction Mockups

Six visual approaches for the Todo app. Same color palette, same type scale — different layout, density, and interaction character.

Direction 1 — Stark Minimal

Underline input, no surface elements, maximum negative space. Every pixel earns its place.

things to do
  • Research flight options to Lisbon
  • Buy new sketchbook — Moleskine A5
  • Send project proposal to Elena
  • Schedule dentist appointment
  • Update portfolio site

Design Character

  • Underline-only input — dissolves into the page when empty
  • No background surfaces, no cards — just text and lines
  • Delete action revealed on hover as text ("remove"), not an icon
  • Completed items dimmed inline with strikethrough
  • Maximum reliance on negative space for hierarchy

Direction 2 — Notebook Lines

Ruled-line background evokes the physical notepad. Tasks sit on lines like handwriting.

Design Character

  • Repeating horizontal lines create a ruled-notebook feel
  • Small dot indicators instead of checkboxes — less visual noise
  • Tasks aligned to the ruled grid like handwritten notes
  • Centered title with letter-spacing for a cover-page feel
  • Most literal interpretation of the Moleskine mental model

Direction 3 — Elevated Cards

Each task is a distinct card with subtle depth. More tactile, more structured.

Design Character

  • Each task is a distinct card with border, border-radius, and surface color
  • Hover lifts the card with translateY and shadow — tactile feel
  • Gap between cards (8px) creates clear separation
  • Timestamps visible — adds context without clutter
  • More structured feel — each task feels like a discrete object

Direction 4 — Dense & Functional

Compact layout with checkboxes and task count. Efficient, tool-like, no wasted space.

  • Buy new sketchbook — Moleskine A5 2h
  • Send project proposal to Elena 1d
  • Schedule dentist appointment 1d

Design Character

  • Smaller font size (14px) — fits more tasks in viewport
  • Checkbox squares — explicit completion affordance
  • Task count in header provides context at a glance
  • Abbreviated timestamps save horizontal space
  • Hover reveals row background rather than text color change
  • Feels tool-like — efficiency over elegance

Direction 5 — Bottom Input

Input anchored to the bottom. The list is the hero — you add to it from below.

  • Research flight options to Lisbon
  • Buy new sketchbook — Moleskine A5
  • Send project proposal to Elena
  • Schedule dentist appointment

Design Character

  • Input at the bottom — like adding a line at the end of a notebook page
  • The existing list dominates the view — new tasks feel like additions
  • Mimics chat/messaging mental model — familiar pattern
  • Title and list are first things you see — input is secondary
  • Same stark aesthetic as Direction 1 but with reversed spatial hierarchy

Direction 6 — Typographic

Editorial, magazine-like. Left accent bar, larger text, italic completions. Type as art.

  • Send project proposal to Elena yesterday

Design Character

  • Left accent bar on input — editorial marker, draws the eye
  • Larger text (18px) with light weight (300) — elegant, readable
  • Completed tasks use italic + strikethrough — not just dimmed
  • Hover reveals left border on each item — subtle interactive cue
  • Subtitle adds personality — "a simple list"
  • Uppercase metadata with letter-spacing — magazine typography
  • Most editorial, least tool-like direction
